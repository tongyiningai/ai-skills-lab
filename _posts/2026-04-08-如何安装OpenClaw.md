定位远程构建日志
问题的根源很可能藏在远程的构建日志里。
进入 Actions 页面：在你的 GitHub 仓库页面，点击上方的 Actions 标签。
找到失败的工作流：在左侧的 All workflows 列表中，找到名为 pages-build-deployment 或 Build and Deploy 的工作流，并点击进入。
查看详细日志：选择状态为 ❌ 的最新一次运行记录，点击进入后，在 build 或 deploy 的步骤下展开日志，仔细查找具体的报错信息。

检查 GitHub Pages 部署的源（最关键）
进入你的仓库：Settings → Pages → Build and deployment。
Source 应该设置为 GitHub Actions（而不是 Deploy from a branch）。
如果这里设置的是 Deploy from a branch，那么 Pages 会直接从某个分支（如 main）读取文件，而不会使用 Actions 构建的 _site 内容。这会导致显示原始文件（比如看到 --- layout: home --- 文本）。
